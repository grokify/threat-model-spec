## [Unreleased]

### Highlights

- Versioned specification following OpenAPI pattern (`versions/v0.4.0/`)
- Comprehensive security lifecycle support with mitigations, detections, and response actions
- Attack tree diagrams with AND/OR logic gates for hierarchical threat decomposition
- LINDDUN privacy threat framework with D2 rendering
- Control framework mappings: NIST CSF, CIS Controls v8, ISO 27001
- Compliance framework references: SOC 2, PCI-DSS, HIPAA, GDPR, FedRAMP, and more

### Added

- Attack tree diagram type with AND/OR logic gates (`attack-tree`)
- `AttackTreeNode` and `AttackTree` types for hierarchical attack decomposition
- LINDDUN privacy threat framework (`ir/linddun.go`) with 7 threat categories (L, I, N, D, Di, U, Nc)
- Mitigations tracking (`ir/mitigations.go`) with status lifecycle (planned, implemented, partial, accepted, transferred)
- Threat status tracking with `ThreatEntry` and `ThreatStatus` types
- Control framework mappings (`ir/controls.go`):
  - NIST CSF with 5 functions (Identify, Protect, Detect, Respond, Recover)
  - CIS Controls v8 with safeguards and implementation groups
  - ISO 27001 with control domains
- Compliance framework references (`ir/compliance.go`): SOC 2, PCI-DSS, HIPAA, GDPR, CCPA, FedRAMP, NIST SP 800-53, StateRAMP, ITAR, CMMC
- Threat actor profiles (`ir/threat_actor.go`) with sophistication, motivation, and resources
- Detection capabilities (`ir/detection.go`) with coverage levels and data sources
- Response actions for incident response workflows
- Assumptions and prerequisites (`ir/assumptions.go`) for documenting model constraints
- CVE references in mappings (`CVEMapping` type)
- D2 legend rendering for LINDDUN threats and mitigation status
- Attack tree D2 rendering with node styling and AND/OR gate visualization
- STIX 2.1 export for CVE (Vulnerability), OWASP (AttackPattern), and mitigations (CourseOfAction)
- Comprehensive validation for mitigations, threats, detections, and response actions
- Strict mode validation for recommended fields (owner, severity, data sources)
- Versioned specification structure (`versions/v0.4.0/`) following OpenAPI pattern
- Human-readable specification (`versions/v0.4.0/specification.md`)
- Versioned schema URLs with `$id` pointing to versioned location

### Changed

- `Mappings` struct extended with CVE, LINDDUN, Controls, and Compliance fields
- `DiagramIR` extended with AttackTree, Threats, Mitigations, Detections, ResponseActions fields
- `Legend` extended with ShowLINDDUN and ShowMitigations options
- JSON schemas regenerated to include all new types
- README updated with new framework tables and usage examples
