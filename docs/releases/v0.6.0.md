# v0.6.0 Release Notes

**Release Date:** 2026-04-28

This release introduces comprehensive security enhancements including role-based guidance, risk quantification, threat intelligence, purple team capabilities, supply chain security, and attack path analysis.

## Highlights

- Role-based security data for Red Team, Blue Team, and Remediation workflows
- FAIR risk assessment with annualized loss expectancy calculations
- Enhanced STIX 2.1 export with IOCs, detection rules, and threat actors
- Purple team capabilities with Atomic Red Team mapping and detection coverage matrix
- Security metrics tracking (MTTD, MTTR, detection rates)
- Supply chain security with SBOM integration and VEX statements
- SSVC vulnerability prioritization with automated decision trees
- Attack graph analysis with path finding and risk calculation algorithms
- OWASP Agentic Security Top 10 (ASI 2026) support

## What's New

### OWASP ASI Support

Support for the new OWASP Agentic Security Top 10 (2026) for AI agent threats:

| Category | Value | Description |
|----------|-------|-------------|
| API Security | `api` | API Security Top 10 (2023) |
| LLM Applications | `llm` | LLM Application Top 10 (2025) |
| Web Applications | `web` | Web Application Top 10 (2021) |
| Agentic Applications | `agentic` | Agentic Top 10 (ASI 2026) |

Attack steps can now include `asiIds` for OWASP Agentic Security mappings:

```json
{
  "step": 6,
  "from": "gateway",
  "to": "agent",
  "label": "Access agent",
  "asiIds": ["ASI02:2026", "ASI03:2026"],
  "owaspIds": ["API8:2023"]
}
```

Reference data is available for all 40 OWASP entries via `GetOWASPEntry()` and `ValidateOWASPID()`.

### Role-Based Security Guidance

#### Red Team

Exploitation guidance for penetration testers:

```json
{
  "redTeam": {
    "prerequisites": ["Target must have WebSocket gateway enabled"],
    "exploitationSteps": [
      {"order": 1, "action": "Host malicious page", "tool": "Python HTTP server"}
    ],
    "tools": [{"name": "Burp Suite", "purpose": "WebSocket interception"}],
    "difficulty": "low"
  }
}
```

#### Blue Team

Defense guidance with detection rules:

```json
{
  "blueTeam": {
    "detectionRules": [
      {
        "name": "Cross-Origin WebSocket",
        "format": "sigma",
        "rule": "title: Cross-Origin WebSocket Connection..."
      }
    ],
    "iocs": [
      {"type": "pattern", "value": "ws://localhost:*", "confidence": 0.9}
    ],
    "huntingQueries": [
      {"name": "Find WS connections", "language": "splunk", "query": "..."}
    ]
  }
}
```

#### Remediation

Developer guidance for fixing vulnerabilities:

```json
{
  "remediation": {
    "codePatterns": [
      {

```

### Attack Path Analysis

Graph-based attack surface analysis:

- `FindAllPaths()` — Complete path enumeration with cycle detection
- `FindShortestPath()` — Dijkstra's algorithm for minimal path
- `FindCriticalPaths()` — Highest risk path identification
- `CalculatePathRisk()` — Path-based risk scoring
- `ReachabilityAnalysis()` — Exposure assessment from entry points
