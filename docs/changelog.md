## [v0.6.0] - 2026-04-28

### Highlights

- Comprehensive security enhancement release with role-based guidance, risk quantification, and threat intelligence features
- Role-based security data for Red Team, Blue Team, and Remediation workflows
- FAIR risk assessment with annualized loss expectancy calculations
- Enhanced STIX 2.1 export with IOCs, detection rules, and threat actors
- Purple team capabilities with Atomic Red Team mapping and detection coverage matrix
- Security metrics tracking (MTTD, MTTR, detection rates)
- Supply chain security with SBOM integration and VEX statements
- SSVC vulnerability prioritization with automated decision trees
- Attack graph analysis with path finding and risk calculation algorithms

### Added

#### Section A: OWASP ASI Support

- `OWASPCategoryAgentic` constant for OWASP Agentic Security Top 10 (ASI 2026)
- `ASIIds []string` field on Attack struct for agentic threat mappings
- OWASP reference data for all 40 entries across API, LLM, Web, and ASI lists
- `GetOWASPEntry()` and `ValidateOWASPID()` helper functions
- `ValidateOWASPMappings()` method for validation warnings

#### Section B: Role-Based Security Data

- `ir/redteam.go` with `ExploitationGuidance`, `ExploitationStep`, `OffensiveTool`, `PayloadPattern` types
- `ExploitDifficulty` enum (trivial, low, medium, high, expert)
- `ir/blueteam.go` with `DefenseGuidance`, `DetectionRule`, `IOC`, `HuntingQuery`, `AlertThreshold` types
- `DetectionFormat` enum (sigma, yara, splunk, elastic, kql, snort, suricata, custom)
- `IOCType` enum (ip, domain, url, hash, filepath, email, registry, process, certificate, pattern)
- `ir/remediation.go` with `RemediationGuidance`, `CodePattern`, `ChecklistItem`, `Library`, `ConfigChange` types
- `ir/playbook.go` with `IncidentPlaybook`, `PlaybookStep`, `Contact` types
- `PlaybookPhase` enum for IR phases (preparation, identification, containment, eradication, recovery, lessons-learned)
- `ir/testref.go` with `TestReference`, `TestSuiteReference` for app-test-spec integration
- `RedTeam`, `BlueTeam`, `Remediation`, `Playbooks` fields on ThreatModel struct
- `RedTeamNotes`, `BlueTeamNotes`, `RemediationNote` fields on Attack struct

#### Section C: Risk Quantification

- `ir/fair.go` with `FAIRAssessment`, `FrequencyEstimate`, `LossEstimate`, `Currency` types
- `CalculateALE()` function for Annualized Loss Expectancy calculations
- `ir/epss.go` with `EPSSData` struct for FIRST EPSS scoring
- `BusinessImpact` struct with revenue, customer, regulatory, reputation, operational impact fields
- `Criticality` enum (critical, high, medium, low)
- `RiskAssessment`, `BusinessImpact`, `EPSSData` fields on ThreatModel struct

#### Section D: Threat Intelligence

- `ir/stix_export.go` with enhanced STIX 2.1 export functions
- `IOC.ToSTIXIndicator()`, `ThreatActor.ToSTIXThreatActor()`, `Attack.ToSTIXAttackPattern()` methods
- `DetectionRule.ToSTIXCourseOfAction()` method
- `ThreatModel.ExportSTIXBundle()` method for complete bundle generation
- `ir/kev.go` with `KEVEntry`, `GetKEVEntry()`, `IsInKEV()` for CISA KEV catalog integration

#### Section E: Purple Team

- `ir/atomicredteam.go` with `AtomicTestMapping` struct for Atomic Red Team integration
- `AtomicTestResult` enum (passed, failed, not-run, blocked, error)
- MITRE technique ID format validation
- `ir/coverage.go` with `DetectionCoverageMatrix`, `TechniqueCoverage`, `CoverageSummary` types
- `CoverageLevel` enum (full, partial, none, planned)
- `CalculateCoverage()` function for coverage percentage calculations
- `AtomicTests`, `DetectionCoverage` fields on ThreatModel struct
- `AtomicTestID` optional field on Attack struct

#### Section F: Security Metrics

- `ir/metrics.go` with `SecurityMetrics`, `MetricDuration` types
- `MetricTimeUnit` enum (seconds, minutes, hours, days)
- MTTD, MTTR, MTTC (Mean Time to Detect/Respond/Contain) fields
- `DetectionRate`, `FalsePositiveRate` fields for effectiveness tracking
- `Metrics` field on ThreatModel struct

#### Section G: Supply Chain Security

- `ir/sbom.go` with `SBOMReference`, `ComponentReference`, `DependencyRisk` types
- `SBOMFormat` enum (cyclonedx, spdx)
- `CalculateDependencyRiskLevel()` function for severity-based risk assessment
- `ir/vex.go` with `VEXStatement`, `VEXDocument` types for vulnerability exploitability
- `VEXStatus` enum (not_affected, affected, fixed, under_investigation)
- `VEXJustification` enum for not_affected reasons
- Helper functions: `NewVEXDocument()`, `NewNotAffectedStatement()`, `NewAffectedStatement()`
- `SBOM`, `VEXStatements`, `DependencyRisks` fields on ThreatModel struct
- `ComponentRefs` field on Attack struct for supply chain threat linking

#### Section H: Vulnerability Management

- `ir/ssvc.go` with `SSVCAssessment` struct for CISA SSVC decision trees
- `SSVCExploitation` enum (none, poc, active)
- `SSVCAutomatable` enum (no, yes)
- `SSVCTechnicalImpact` enum (partial, total)
- `SSVCMissionPrevalence` enum (minimal, support, essential)
- `SSVCPublicWellBeing` enum (minimal, material, irreversible)
- `SSVCDecision` enum (track, track_star, attend, act)
- `CalculateSSVCDecision()` function implementing CISA prioritization tree
- `SSVCAssessment` field on ThreatEntry struct

#### Section I: Attack Path Analysis

- `ir/attackgraph.go` with `AttackGraph`, `GraphNode`, `GraphEdge` types
- `GraphNodeType` enum (element, threat, control, asset, actor)
- `GraphEdgeType` enum (flow, attack, mitigation, threat, access)
- `NewAttackGraph()`, `AddNode()`, `AddEdge()`, `GetNode()`, `GetOutgoingEdges()` methods
- `BuildAttackGraphFromDiagram()` function for automatic graph construction
- `ir/attackpath.go` with `AttackPath`, `PathAnalysisResult` types
- `FindAllPaths()` with DFS and cycle detection
- `FindShortestPath()` using Dijkstra's algorithm
- `FindCriticalPaths()` for highest-risk path identification
- `CalculatePathRisk()` for path-based risk scoring
- `ReachabilityAnalysis()` for exposure assessment
- `AnalyzePaths()` for comprehensive path analysis

### Changed

- JSON schemas regenerated for v0.6.0 with all new types
- `Attack` struct extended with role-based notes, atomic test fields, and ComponentRefs
- `ThreatModel` struct extended with 15+ new optional sections including SBOM, VEX, and metrics
- `ThreatEntry` struct extended with SSVCAssessment for vulnerability prioritization
- README updated with v0.6.0 feature documentation including supply chain and attack path features

### Tests

- Unit tests for all new types with JSON serialization round-trip verification
- `sbom_test.go` and `vex_test.go` for supply chain security
- `ssvc_test.go` for SSVC decision tree validation
- `attackgraph_test.go` and `attackpath_test.go` for graph algorithms
- `threat_model_test.go` with ThreatModel validation and v0.6.0 feature tests
- `validate_owasp_test.go` for OWASP mapping validation
- `validate_attack_tree_test.go` for attack tree validation
- Test coverage improved from 54.6% to 66.4% in ir package
